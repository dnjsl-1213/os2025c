리눅스 명령어
- ls : 현재 폴더에 있는 파일 표시
- ls -a : 숨겨진 파일까지 표시 ex) .git . gitignore .vscode
- touch 파일명 : 파일 생성
- cd 폴더명 : 특정 폴더명으로 경로 이동
- cd .. : 상위 디렉토리로 이동
- cat 파일명 : 파일에 내용을 표시

깃 명령어
- git init : 현재 작업중인 디렉토리를 git 저장소로 초기화(폴더 생성 시 가장 처음에 해야하는 git 명령어)
- git status : 파일의 상태를 확인
- git add 파일명 : 디렉토리에 변경사항을 스테이징 영역에 추가
- git add . : 변경된 모든 디렉토리를 스테이징 영역에 추가
- git config --global user.email "개인이메일주소" : git으로 작업하는 모든 프로젝트에 대한 커밋 작성자 이메일 주소 설정
- git config --global user.name "필명" : 이름 설정
- git config --list : 설정된 정보 확인
- git commit -m "커밋에 남길 메세지" : 파일 및 폴더의 변경사항을 git에 저장하기 위한 메세지 작성 과정
- git log : 과거 커밋에 남겨놓은 메세지 확인
- git checkout 해시코드전체 또는 7자리 : 해당 해시코드 브랜치 전환
- git checkout - : 이전에 작업하던 브랜치로 전환

용어
- 디렉토리 : 폴더
- 스테이징 영역 : 디렉토리와 저장소 사이의 영역, 파일 수정 후 git add로 올리면 그 파일은 다음 git commit 명령어를 위한 준비상태가 됨
- 저장소 : git과 같은 버전 관리 시스템에서 사용되는 용어
- 브랜치 : 독립적인 작업 공간(원본(main)에 영향을 주지 않아 다른 브랜치로 로그인하여 작업 후 개발이 완료되면 main에 변경사항을 저장하는 용도)
